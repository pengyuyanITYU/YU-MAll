# 1. 修改 .gitignore
在 `.gitignore` 文件末尾追加 `**/target/` 规则，以显式忽略所有子目录下的 `target` 文件夹。这一步也满足了“修改 .gitignore 文件”的要求。

# 2. 从 Git 索引中移除 target
针对所有存在的 `target` 目录执行 `git rm -r --cached` 命令。
由于项目包含多个子模块，将通过脚本遍历或批量执行命令，确保覆盖所有 `target` 目录（如 `address-service/target`, `cart-service/target` 等）。
命令将包含 `--ignore-unmatch` 参数以避免因文件未被跟踪而报错。

# 3. 验证状态
运行 `git status` 检查结果。
预期输出：
- `.gitignore` 显示为已修改（modified）。
- `target` 目录下的文件显示为已删除（deleted），且不再出现在 untracked 列表中（因为被忽略了）。
